# 식집사 다국어화(i18n) 설계 — 한국어·영어·일본어

작성일: 2026-06-15

## 목표
앱을 **한국어(ko)·영어(en)·일본어(ja)** 3개 언어로 사용 가능하게 한다. 기기 언어 자동 감지 + '나' 탭에서 수동 전환(저장). 일본어는 정중체(です/ます).

## 결정 사항 (사용자 승인)
- 번역문은 **전부 내가 작성**(영·일). 일본어 です/ます 톤.
- 언어 선택: **기기 자동감지 + 수동 전환**, AsyncStorage 저장. 미지원 로케일 → 영어 fallback.
- 약관/개인정보 웹페이지도 **영·일 버전** 제작, 앱 언어에 맞게 연결.
- 사용자 입력 데이터(식물 이름·메모·공간명)는 번역 대상 아님.
- **코드 주석의 한글은 번역하지 않음** — 사용자 노출 문자열만 추출.

## 라이브러리
- `i18next` + `react-i18next` (컴포넌트 `useTranslation()` / 비-컴포넌트 `i18n.t()`)
- `expo-localization` (기기 로케일 감지)
- `@react-native-async-storage/async-storage` (이미 설치됨, 선택 언어 저장)

## 파일/디렉토리 구조
```
src/i18n/
  index.ts            # i18next 초기화, 기기감지, 저장/복원, changeLanguage 래퍼
  useLanguage.ts      # { language, setLanguage, available } 훅
  locales/
    ko/<namespace>.json
    en/<namespace>.json
    ja/<namespace>.json
```
- **언어 × 네임스페이스 = 파일 1개**. 병렬 작업 시 충돌 방지(각 에이전트가 자기 네임스페이스 파일만 씀).
- `index.ts`는 모든 네임스페이스를 정적 import해 등록(아래 네임스페이스 목록 고정).

## 네임스페이스 & 파일 분할 (병렬 추출 단위)
각 그룹 = 1 네임스페이스 = 1 에이전트가 담당. **에이전트는 자기 그룹 파일만 편집**하고 `locales/{ko,en,ja}/<ns>.json`만 생성/수정한다. 그룹 간 파일 겹침 없음.

## 키 네이밍 규칙
- `t('<ns>.<camelCaseKey>')`, 예: `t('auth.kakaoButton')`, `t('home.todayTasksEmpty')`.
- 변수 삽입: `t('weather.updatedHoursAgo', { hours })` → `"{{hours}}시간 전 업데이트"`.
- 복수형(영어만 구분): i18next 복수 규칙 사용. 키는 `count` 전달. ko/ja는 단일 형태.
  - 예: `plantCount` → ko `"식물 {{count}}개"` / en `_one`"{{count}} plant"`_other`"{{count}} plants"` / ja `"植物{{count}}個"`.
- 공용 단어는 새 키 만들지 말고 `common.*` 재사용(취소=`common.cancel` 등).

## 언어 선택 동작
- 시작: 저장된 `app.language` 있으면 사용 → 없으면 `expo-localization` 기기 언어를 ko/en/ja로 매핑(그 외 en).
- '나' 탭 **"언어 · Language"** 섹션: `한국어 / English / 日本語` 세그먼트(기존 테마 세그먼트 디자인 재사용). 선택 즉시 `i18n.changeLanguage` + 저장.
- 언어 변경 시 **예약 알림 재등록**(기존 reminder-hour 변경 시 rescheduleAll 경로에 연결).

## 약관/개인정보 다국어 (gh-pages)
- `terms.html`(ko, 기존)·`terms.en.html`·`terms.ja.html`, `privacy.*` 동일. 각 페이지 상단 언어 전환 링크.
- 앱 `src/lib/legal.ts`의 `LEGAL_URLS`를 현재 언어 인자로 받아 해당 URL 반환하도록 변경.

## 실행 단계 (병렬화 지점 표시)
0. **[순차·내가] 토대**: 의존성 설치, `src/i18n/index.ts`·`useLanguage.ts` 작성, `common` 네임스페이스 시드, 빈 네임스페이스 파일 스캐폴드, `app/_layout.tsx` 초기화 연결, '나' 탭 언어 선택 UI. → typecheck/번들 통과 확인.
1. **[병렬·에이전트] KO 추출**: 네임스페이스 그룹별 에이전트가 자기 파일의 한글 UI 문자열 → `t()` 치환 + `locales/ko/<ns>.json` 작성.
2. **[병렬·에이전트] 번역**: (언어 × 네임스페이스)별로 `locales/{en,ja}/<ns>.json` 작성(ko 기준 번역).
3. **[순차·내가] 약관 페이지 영·일 + legal.ts 언어 연결.**
4. **[순차·내가] 검증**: 잔존 한글 UI 문자열 grep 검사(로케일/주석 제외), typecheck, `expo export` 번들, 세 언어 수동 전환 점검.

## 검증 기준
- `tsc --noEmit` 통과, `expo export --platform ios` 번들 성공.
- 각 화면에서 ko/en/ja 전환 시 텍스트가 모두 바뀌고 깨진 키(`ns.key` 노출) 없음.
- 잔존 추출 누락 검사 스크립트로 사용자 노출 한글 0건(주석·데이터 예외 처리).
